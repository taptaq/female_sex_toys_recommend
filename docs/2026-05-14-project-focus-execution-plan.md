## 一、总目标

未来一段时间的产品优化都围绕这 3 个目标推进：

1. `降低认知负担`
2. `缩短从匹配到购买决策的距离`
3. `提高推荐结果的信任感与行动承接能力`

## 当前状态说明

- `已完成第一版`：主链路目标已落地，但后续仍可能继续做组件拆分、文案收口、信息层级微调
- `持续打磨`：功能方向已成立，当前重点是减重、稳态和可维护性，而不是从零补功能
- `冻结观察`：短期不继续加重投入，只保留现状并观察真实使用反馈

当前整体判断：

- `P0`：已基本完成第一版
- `P1`：已基本完成第一版
- `P2`：按计划冻结观察，不作为当前继续扩重的目标

## 二、执行原则

### 1. 先改主链路，不先修边缘模块

优先顺序固定为：

1. 首页
2. 匹配问答
3. 结果页
4. 结果后的行动承接
5. 深挖层模块

### 2. 先做减法，再做增强

如果一个功能：

- 不能明显减少用户犹豫
- 不能明显帮助用户更快做决定
- 不能明显增强结果信任感

那它不应该排在当前优先级前列。

### 3. 结果页不再承担“宇宙总控台”职责

结果页只优先回答：

- 推荐哪一类
- 为什么
- 什么不要优先看
- 下一步怎么办

其他内容全部后置。

### P0-2 问答“不确定”分支补齐与校准

> 当前状态：**已完成第一版，不是从零开始。**
>
> 现有女性 / 男性 / 情侣主链路中的多数关键题，已经提供了
> `先帮我判断 / 还不确定 / 先按稳妥判断`
> 一类选项，并已接入保守兜底评分逻辑。
>
> 本任务的重点不再是“有没有不确定分支”，而是：
> `是否覆盖完整`、`兜底是否足够稳`、`文案是否足够自然`。

#### 目标

降低新手被迫硬选的情况，减少“我其实不懂但只能瞎选”的失真。

#### 任务

- 盘点现有关键题里的“不确定”覆盖范围，确认是否还有遗漏
- 校准不同分支下的保守默认策略，避免兜底仍把用户拉向过强或过窄路线
- 统一“不确定”选项文案，减少：
  - `我不确定`
  - `都可以`
  - `先帮我判断`
  - `先按稳妥判断`
  之间的表达割裂
- 明确哪些题应当继续要求明确选择，哪些题适合保留“不确定”
- 避免“不确定”变成没有意义的占位项，必须持续参与判定

#### 优先题型

- 刺激路线
- 强度偏好
- 外部 / 入体 / 双刺激
- 隐蔽优先还是体验优先
- 互动方式 / 双方偏好 / 共玩场景

#### 建议涉及范围

- 问题定义文件
- 分支逻辑文件
- 问答相关测试

#### 验收标准

- 新手可在“不知道”的前提下继续完成问答
- 不确定答案不会直接把结果拉偏到极端路线
- 关键题的“不确定”文案风格一致
- 仍需明确选择的题目边界清晰：
  - 例如首题 `航向选择` 继续要求用户明确方向

## 六、每个任务的统一验收问题

每个任务上线前都问一次：

1. 它是否让用户更快做决定？
2. 它是否减少了结果页或首页的认知负担？
3. 它是否让知识、装备库、人格测试更像“服务主决策”的模块，而不是平行产品？
4. 它是否引入了新的复杂度，却没有带来明确决策收益？

如果第 4 条答案是“是”，就应该降级优先级。
