# Luna 女性向装备推荐 MVP

Luna 是一个移动端优先的女性向私密装备推荐 App MVP。

它的核心不是把用户直接丢进商品库，而是用一段轻量问答，把预算、场景、隐私、清洁、新手友好度和感受偏好转成可解释的推荐结果。视觉上以「萌系宇航员 IP」和「星球采集线索」作为主叙事：Luna 先去不同星球收集偏好，最后帮用户解锁更适合的装备。

## 当前定位

- 面向女性用户的装备推荐体验。
- 移动端一屏优先，减少滚动、减少复杂入口。
- 首页强调 Luna IP、柔和星际感、女性向安全感和轻量决策。
- 问答和结果页服务「帮我更安心地选」这个主任务。
- 装备库、知识星云、匹配档案保留为次级承接，不抢首页主链路。

## 核心体验

1. 首页

   Luna 小宇航员和四颗核心星球组成 3D 轮播式探索动画，最终收束到「装备舱」状态，再展示标题、信任标签和主按钮。

2. 答题页

   用女性向、场景化的问题收集偏好，避免一开始就让用户面对复杂参数。

3. 匹配加载页

   用 Luna 和星球动效承接等待过程，让「正在匹配」更像装备校准，而不是普通 loading。

4. 结果页

   先给出主推荐和简短解释，再展示备选说明、注意事项、微调入口和可继续比较的商品信息。

5. 装备库与知识模块

   作为推荐后的补充比较与解释层，帮助用户理解材质、清洁、静音、防水、收纳等关键判断点。

## 快速开始

安装依赖：

```bash
npm install
```

启动前端：

```bash
npm run dev
```

默认访问：

```text
http://127.0.0.1:3009
```

启动服务端：

```bash
npm run server
```

前后端一起启动：

```bash
npm run dev:all
```

只查看前端交互时，通常先跑 `npm run dev` 即可。涉及推荐生成、数据库、AI 重排、档案保存和反馈提交时，需要配置对应 `.env`。

- `src/pages/HomePage.tsx`：Luna 首页、首页动画和入口体验。
- `src/pages/QuestionnairePage.tsx`：答题页。

## 推荐链路

推荐结果由本地结构化筛选和 AI 理由增强共同完成：

- 问卷答案会转成结构化偏好信号。
- 本地先根据预算、适用对象、静音、防水、材质、收纳、清洁和形态等维度筛选候选。
- AI 对候选进行重排、解释和购买建议补充。
- 结果页支持轻量微调，比如更安静、更适合新手、预算更低。
- 推荐会话可写入 `recommendation_sessions`，便于后续回放、评测和人工校准。

## 测试重点

- 首页主入口保持轻量，不能重新暴露过多次级入口。
- 首页动画比例、星球数量、移动端一屏布局需要被测试保护。
- 答题页的每个选项应能影响推荐信号，而不是只做展示。
- 结果页背景需要铺满，避免出现黑色边框或桌面式卡片外壳。
- 推荐评测应覆盖女性新手、隐私敏感、静音、清洁和护理周边排除等场景。

## 设计方向

整体风格关键词：

- 女性向
- 萌系宇航员
- 柔软星际
- 奶油蓝粉
- 轻盈玻璃质感
- 安全、私密、被照顾
- 高级但不冷淡

交互上优先「丝滑、轻、像在被引导」，避免普通电商感、参数控制台感和过重的游戏化装饰。

## 当前取舍

这个版本优先打磨首页、答题、匹配加载和结果页主链路。装备库、知识星云、档案、爬虫和数据库清洗能力仍在项目里，但它们不是 MVP 首页的主角。

后续如果继续扩展，建议先验证：

- 首页首屏是否足够吸引目标女性用户。
- 用户是否能快速理解「找到装备」而不是「找到星球」。
- 结果页是否能让用户知道下一步该先比较什么。
- 推荐理由是否足够具体、安心、不过度营销。
